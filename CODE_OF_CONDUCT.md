# Code of Conduct

honeyeater is a company-maintained project. Participation in its repositories is expected to be professional: be civil, be precise, critique code and arguments rather than people.

The maintainers curate these spaces at their discretion using GitHub's standard tools — hiding or removing content, locking threads, and blocking accounts where conduct falls short of that expectation. [GitHub's Acceptable Use Policies](https://docs.github.com/en/site-policy/acceptable-use-policies/github-acceptable-use-policies) apply to all activity on the platform independently of this document.

Concerns about conduct in project spaces can be raised with the maintainers through the repository's issue tracker.
